# [TRD] 빈티지 카페 '주례 브루어스 (Jurye Brewers)' 기술 요구사항 문서

## 1. 시스템 아키텍처 개요 (System Architecture Overview)
* **개발 방식:** 노코드/로우코드 SaaS 플랫폼 (추천: 아임웹 또는 식스샵)
* **인프라:** 해당 SaaS 제공업체의 완전 관리형 클라우드 호스팅 및 CDN 이용
* **데이터베이스:** SaaS 플랫폼 내장 데이터베이스 (회원 정보, 상품 정보, 주문 데이터 관리)

## 2. 외부 연동 및 API 요구사항 (External Integrations & APIs)

### 2.1. 인증 및 회원 관리 (IAM)
* **OAuth 2.0 연동:** 카카오 로그인 API, 네이버 아이디로 로그인 API, 구글 로그인 API
* **요구 기능:** 최초 로그인 시 회원가입 및 약관 동의 프로세스 자동 연동

### 2.2. 결제 시스템 (PG)
* **PG사 연동:** 토스페이먼츠, KG이니시스, 나이스페이먼츠 중 택일 연동
* **간편결제 연동:** 카카오페이, 네이버페이, 토스페이 활성화
* **필수 보안 사양:** SSL/TLS 보안 인증서 적용 및 데이터 암호화 전송

### 2.3. 지도 및 위치 정보
* **지도 API:** 네이버 지도 API (또는 카카오 맵 API) 웹 컴포넌트 임베드
* **기능:** 매장 위치 핀 고정, 모바일 터치 시 네이버 지도 앱/카카오내비 연동 길찾기 기능 지원

## 3. 프론트엔드 및 인터페이스 요구사항 (Frontend Requirements)
* **반응형 디자인 (Responsive Web):** PC, 태블릿, 모바일 해상도 전용 뷰포트 최적화 (Mobile-First 대응)
* **폰트 및 스타일링:** 웹 안정형 빈티지 감성 폰트(Noto Serif KR 등 명조 계열 또는 프리미엄 라이선스 폰트 적용)
* **미디어 최적화:** 이미지 인풋 시 WebP 포맷 또는 리사이징을 통한 초기 페이지 로딩 속도 최적화 (2초 이내 타겟)

## 4. 운영 및 백오피스 요구사항 (Operations & Admin)
* **상품 관리:** 원두 용량별(예: 200g, 500g) 옵션가 설정 기능
* **주문/배송 관리:** 운송장 번호 대량 입력 및 배송 상태(배송중, 배송완료) 자동 알림톡 기능 연동
* **보안 및 규정:** 개인정보 처리방침 고지 및 쇼핑몰 하단 하이퍼링크 연결 (국내 전자상거래법 준수)
